# Portfolio CMS

A complete, production-ready Professional Portfolio Website CMS built with **Laravel 12**, **MySQL**, **Bootstrap 5**, and a custom-built admin dashboard. Manage your About info, Skills, Services, Experience, Education, Portfolio Projects, Blog, Testimonials, Contact Messages, Site Settings, and SEO — all from one clean admin panel.

---

## Tech Stack

- Laravel 12 / PHP 8.2+
- MySQL
- Bootstrap 5 (CDN)
- Blade Templates
- Laravel built-in Authentication
- Spatie Laravel Permission (roles & permissions)
- CKEditor 5 (CDN)
- DataTables (CDN)
- SweetAlert2 (CDN)
- Font Awesome 6 (CDN)
- UniSharp Laravel File Manager + Intervention Image

> **Note:** Bootstrap, Font Awesome, CKEditor, DataTables, jQuery, SweetAlert2, and Chart.js are all loaded from CDNs — no `npm install` / asset build step is required to view the site.

## Important — Before You Start

This project's PHP source code (controllers, models, migrations, Blade views, routes, seeders) is complete and ready to use. However, like every Laravel project, it needs its **`vendor/` directory** generated by Composer — this repository does not (and should not) ship `vendor/` pre-built. You must run `composer install` as part of setup (Step 4 below). This is standard for any Laravel project and not specific to this CMS.

## Installation Guide

### 1. Place the Project

Copy/extract the project so that it lives at:

```
C:\xampp\htdocs\portfolio-cms
```

### 2. Create the Database

Open phpMyAdmin (or any MySQL client) and create a database:

```
Database Name: portfolio_cms
```

You do **not** need to import any SQL manually — Step 5 below creates all tables automatically. (An optional hand-written schema-only SQL file is provided at `database/sql/portfolio_cms.sql` if you prefer importing via phpMyAdmin instead of the CLI — see the note at the bottom of that file.)

### 3. Configure Environment

Copy `.env.example` to `.env`:

```bash
copy .env.example .env
```

Open `.env` and confirm the database section matches your local MySQL setup (defaults already match a standard XAMPP install):

```
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=portfolio_cms
DB_USERNAME=root
DB_PASSWORD=
```

### 4. Install Dependencies

Open a terminal in the project folder (`C:\xampp\htdocs\portfolio-cms`) and run:

```bash
composer install
```

This downloads Laravel itself and every package listed in `composer.json` (Spatie Permission, Laravel File Manager, Intervention Image, etc.) into a `vendor/` folder.

### 5. Generate App Key & Run Migrations

```bash
php artisan key:generate
php artisan migrate --seed
php artisan storage:link
```

`migrate --seed` will:
- Create every table (users, projects, blogs, skills, services, experience, education, testimonials, messages, settings, SEO, visitors, roles/permissions, etc.)
- Create the **Admin** and **Editor** roles with the correct permissions
- Create the default admin user (see credentials below)
- Seed realistic demo content for every module so the site looks complete immediately

### 6. Publish the File Manager Assets (one-time)

```bash
php artisan vendor:publish --tag=lfm_public
```

This copies the Laravel File Manager's CSS/JS assets into `public/vendor/laravel-filemanager`.

### 7. Serve the Application

**Option A — XAMPP / Apache (recommended for this setup):**
Make sure Apache & MySQL are running in the XAMPP control panel, then visit:
```
http://localhost/portfolio-cms
```
The included root `.htaccess` automatically forwards all requests to the `public/` folder.

**Option B — Laravel's built-in server:**
```bash
php artisan serve
```
Then visit `http://localhost:8000`.

## What's Included

### Public Website
- Hero section with photo, name, designation, intro, and CTA buttons
- About Me section
- Skills with animated progress bars
- Services grid
- Work Experience & Education timelines
- Portfolio Projects grid with category filtering and a dedicated project detail page (with image gallery)
- Testimonials carousel-style grid
- Blog listing (search + category filter) and full article pages with SEO meta tags

- Fully responsive, "Modern SaaS" themed design using the brand colors:
  - Primary: `#2563EB`
  - Secondary: `#0F172A`
  - Accent: `#F97316`

### Admin Panel (`/admin`)
- Secure login (only users with the `Admin` or `Editor` role can access the panel)
- Dashboard with live stats (Projects, Services, Skills, Testimonials, Blogs, Messages, Visitors) plus visitor trend & browser charts
- Full CRUD for: About, Skills, Services, Experience, Education, Projects (+ categories + gallery images), Blog (+ categories), Testimonials
- Contact Messages inbox (view / mark read / mark unread / delete)
- Site Settings (name, logo, favicon, contact info, social links, Google Map embed)
- SEO Settings (default meta title/description/keywords, Open Graph image)
- User management with role assignment (Admin / Editor)
- Roles & Permissions management (Spatie Laravel Permission)
- DataTables-powered listing tables, SweetAlert2 delete confirmations, CKEditor 5 rich text fields, and a "Insert Image from Gallery" button wired to the Laravel File Manager popup

### REST API (JSON)
```
GET /api/projects
GET /api/projects/{slug}
GET /api/blogs
GET /api/blogs/{slug}
GET /api/services
GET /api/services/{id}
```
(Also available under `/api/v1/...` for versioned access.)

### Security
- CSRF protection on every form (Laravel default)
- XSS protection via Blade's automatic output escaping (`{{ }}`)
- Server-side validation on every form submission
- Route-level authentication + role-based authorization middleware (`auth`, `admin`)
- Password hashing via bcrypt

## Project Structure Highlights

```
app/
  Http/Controllers/
    Admin/      → Admin panel CRUD controllers
    Api/        → REST API controllers
    Auth/       → Admin login/logout
    Front/      → Public-facing site controllers
  Models/       → Eloquent models for every module
  Http/Middleware/
    AdminMiddleware.php   → Restricts /admin to Admin/Editor roles
    TrackVisitor.php      → Lightweight visitor analytics logger

database/
  migrations/   → One migration per table, fully commented
  seeders/      → One seeder per module + master DatabaseSeeder
  factories/    → Factories used by seeders for demo content
  sql/portfolio_cms.sql → Optional manual-import schema (see file header)

resources/views/
  admin/        → Full admin panel Blade views (custom Bootstrap 5 theme)
  front/        → Public-facing Blade views
  components/   → Shared Blade components (e.g. validation errors)

routes/
  web.php       → Public + admin panel routes
  api.php       → REST API routes
```

## Troubleshooting

**"could not find driver" or database connection errors**
Make sure the `pdo_mysql` extension is enabled in your `php.ini` (XAMPP enables this by default) and that MySQL is running in the XAMPP control panel.

**Page is blank / 500 error**
Run `php artisan key:generate` if you haven't already, and check `storage/logs/laravel.log` for the specific error. Also verify `storage/` and `bootstrap/cache/` are writable.

**CSS/JS not loading correctly**
This project intentionally loads Bootstrap, Font Awesome, CKEditor, DataTables, and SweetAlert2 from public CDNs to avoid requiring a Node/npm build step. Confirm your server has internet access, or swap the CDN `<link>`/`<script>` tags in `resources/views/admin/layouts/app.blade.php` and `resources/views/front/layouts/app.blade.php` for local copies if working offline.

**"Class not found" errors after composer install**
Run `composer dump-autoload`.

**Laravel File Manager popup shows an error**
Make sure you ran `php artisan vendor:publish --tag=lfm_public` and `php artisan storage:link`, and that you're logged in as an Admin/Editor (the file manager routes are protected by the same `admin` middleware as the rest of the panel).
